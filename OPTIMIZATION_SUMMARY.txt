🔧 CRITICAL FIX APPLIED

Issue:  Foreign key constraint violations for bill_line_items
Status: ✅ FIXED
File:   HOTFIX_FOREIGN_KEY_DEPENDENCIES.md

The optimized sync service now validates parent record existence before
attempting to sync child records. This prevents foreign key violations
and ensures data integrity.

Affected Tables:
• bill_line_items    → requires parent bills in Supabase
• bill_audit_logs    → requires parent bills in Supabase  
• inventory_items    → requires parent inventory_bills for batch_id

What Changed:
Added explicit checks before syncing child records. If parent doesn't
exist in Supabase, child records are skipped and retried on next sync.
