# Implementation Plan: Country & Multi-Currency Schema Widening

**Branch**: `014-country-currency-schema` | **Date**: 2026-04-21 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification from `/specs/014-country-currency-schema/spec.md`

## Summary

Phase 2 of the multi-currency / country-aware store rollout (`specs/008-multi-currency-country/TASKS.md`, Tasks 3–6). This phase widens the database schema and TypeScript contracts so a `stores` row can declare its `country` and `accepted_currencies` first-class, swaps every legacy `'USD' | 'LBP'` literal union for the `CurrencyCode` type already published by `@pos-platform/shared` in Phase 1, and bumps the local Dexie schema to v55 with a back-fill upgrade. **No runtime behavior changes** — every existing flow keeps working, both apps still compile and run identically. The phase exists only to remove the structural blockers preventing Phases 3–12 from shipping.

## Technical Context

**Language/Version**: TypeScript 5.x, Node.js ≥18
**Primary Dependencies**: Dexie v4 (IndexedDB), Supabase JS v2, React 18, Vite 7, Tailwind CSS 3, Electron 38 (store-app)
**Storage**: Supabase (PostgreSQL — remote, source of record for cross-device); IndexedDB via Dexie v4 (local, primary for store-app)
**Testing**: Vitest (unit + parity gate). No new tests required for type-only widening; existing parity gate must stay green.
**Target Platform**: Web (both apps), Electron NSIS installer (store-app only). Migration runs on Supabase managed Postgres.
**Project Type**: pnpm monorepo — `apps/store-app`, `apps/admin-app`, `packages/shared`
**Performance Goals**: No new performance targets. Migration completes in <2 s on a populated production database; Dexie upgrade hook completes in <1 s on a populated local DB (typical store: ~1 store row, 10²–10³ inventory items).
**Constraints**: Offline-capable (Dexie upgrade must run with no network). Backward-compatible (additive columns with defaults; no breaking schema deletions). Mid-flight clients on the old build must continue to read/write without error.
**Scale/Scope**: ~5 file edits total (1 SQL migration, 1 shared types file, 1 store-app `database.ts`, 1 store-app `types/index.ts`, 1 Dexie `db.ts`). Touches 4 Supabase tables (`stores`, `inventory_items`, `transactions`, `cash_drawer_accounts`) at the type level; 2 tables (`stores`, `inventory_items`) at the Dexie upgrade level.
